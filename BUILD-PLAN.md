## 0. Что решает всё (прочитать первым)

Регламент **отсеивает идеи и макеты.** Проходит только **работающий инструмент, с которым
человек может взаимодействовать** — сайт/бот/приложение. Проекты «только Figma, без кода»
снимаются автоматически на техническом фильтре. Значит:

- Строим **одну маленькую вещь, которая реально работает и задеплоена**, а не «экосистему».
- **История коммитов — это отдельный критерий** (№3). Начать репозиторий **сегодня** и
  коммитить часто, каждый маленький шаг. Код, написанный до 17 июля — дисквалификация; наш
  код весь новый, коммиты с датами 19–21 июля это докажут.
- **Живой деплой на Vercel** = наш «Demo MVP» для сдачи (плюс бэкап — видео до 3 минут).

**Северная звезда (что должно работать на демо):**
> Пользователь пишет: *«Работаю официантом, договор не дали, зарплату задержали на 2 недели»* →
> через несколько секунд получает карточку: **ваше право** (простыми словами) + **статья ТК РК**
> (ссылка на adilet) + **3 конкретных шага сегодня** + **куда обратиться** (1414 / госинспекция

Если это стабильно работает по живой ссылке — мы уже сильнее любого макета в зале.

## 1. Что строим прямо сейчас (точный scope)

### 1.1 Экраны (их всего 1–2 — не усложнять)

1. **Главный экран (чат):**
   - заголовок + одна строка что это («Объясню твои права на первой работе простым языком»);
   - поле ввода: «Опиши свою ситуацию своими словами…»;
   - 3–4 кнопки-примера («Не дали договор», «Не платят переработки», «Задержали зарплату») —
     чтобы человек понял за 5 секунд, о чём продукт (и чтобы демо было гладким);
   - переключатель язык RU / KZ (можно v2, если время);
2. **Карточка-ответ** (появляется под вопросом), структурировано:
   - **Твоё право** — 1–2 предложения простым языком;
   - **Основание** — «Статья N Трудового кодекса РК» + ссылка на adilet.zan.kz;
   - **Что делать (шаги)** — 2–4 конкретных пункта;
   - **Куда обратиться** — 1414 / госинспекция труда / Enbek.kz;
   - **Дисклеймер** — «Это не юридическая консультация, а объяснение и навигация».

### 1.2 Что режем из v1 (явно, чтобы успеть)

Авторизация, сохранение истории, личный кабинет, вся аренда/бизнес/потреб-права (только
первая работа), голосовой ввод, оплата, мобильное приложение/APK. Всё это — «если останется
время» или в питч как «дорожная карта».

**безопасность** (критерий финала №2).

**Ключ API:** нужен аккаунт на platform.claude.com → создать API key → положить в `.env.local`
(локально) и в Environment Variables на Vercel (для деплоя). **Никогда не коммитить ключ.**
Добавить `.env.local` в `.gitignore`.

## 3. Архитектура ИИ — это то, что даёт баллы за «глубину интеграции»

> Критерий №2 прямо говорит: при ИИ ценятся **«кастомные цепочки, а не пустые обёртки над
> API»**. «Пустая обёртка» = просто прокинуть вопрос в ChatGPT. Мы делаем цепочку с
> **обоснованием законом (RAG-подход)** и структурированным ответом. Это наша защищаемая
> инженерная фича.

### 3.1 Цепочка (chain) — рекомендуемый вариант

```
Сообщение пользователя (свободный текст, RU/KZ)
   │
   ├─▶ ШАГ 1 (claude-haiku-4-5): классификация ситуации
   │      → вернуть JSON: { тип_проблемы, есть_ли_договор, релевантные_id_статей: [...] }
   │
   ├─▶ ШАГ 2 (наш код, без ИИ): достать эти статьи из knowledge-base.json
   │      → top-3 релевантных сниппета (текст статьи + номер + ссылка)
   │
   └─▶ ШАГ 3 (claude-sonnet-5): сгенерировать ответ СТРОГО на основе этих статей
          system-промпт: «Отвечай ТОЛЬКО на основе приведённых статей. Всегда указывай
          номер статьи. Если в статьях нет ответа — честно скажи об этом. Простым языком,
          для 18-летнего. Не выдумывай закон.»
          → вернуть структурированный JSON: { право, статьи[], простыми_словами,
             шаги[], куда_обратиться[], дисклеймер }
   │
   ▼
Отрисовать карточку в UI
```

**Почему это сильно и честно:**
- **Grounding (заземление на закон)** — модель отвечает по нашим сниппетам, а не «из головы».
  Это снижает галлюцинации и даёт ссылку на статью → доверие + защищаемость на Q&A.
- **Кастомная цепочка** — классификация → retrieval → генерация со structured output. Это не
  «обёртка», это архитектура. Ровно то, что просит критерий №2.
- **Дешёвый + умный** — дешёвый haiku на подбор, умный sonnet на объяснение. Осознанный
  инженерный выбор (можно объяснить жюри → баллы за защиту).

### 3.2 Упрощённый вариант (если время горит — честный fallback)

Один вызов `claude-sonnet-5`: в system-промпт положить **все ~20 статей** сразу (их немного,
влезут в контекст) + инструкцию «выбери релевантные, ответь строго по ним, укажи статью,
верни JSON». Работает, тоже с ссылками на закон. Минус: чуть дороже за вызов и менее «цепочка».
Начните с этого варианта в первый день (быстрее запустить), потом разбейте на 2 шага (§3.1),
когда будет базовый рабочий поток. **Честно скажите на защите, какой вариант у вас в коде.**

### 3.3 Каркас серверного route (TypeScript, Next.js)

`app/api/ask/route.ts` — упрощённый вариант (§3.2), потом усложняете:

```ts
import Anthropic from "@anthropic-ai/sdk";
import kb from "@/data/knowledge-base.json"; // ~20 статей

const client = new Anthropic(); // читает ANTHROPIC_API_KEY из окружения

const SYSTEM = `Ты помощник по трудовым правам для молодёжи Казахстана.
Отвечай ТОЛЬКО на основе приведённых ниже статей Трудового кодекса РК.
Всегда указывай номер статьи-основания. Пиши простым языком, как для 18-летнего.
Если в статьях нет ответа — честно скажи об этом, не выдумывай закон.
База знаний (статьи):\n${JSON.stringify(kb)}`;

export async function POST(req: Request) {
  const { question } = await req.json();

  const response = await client.messages.create({
    model: "claude-sonnet-5",
    max_tokens: 1500,
    system: SYSTEM,
    output_config: {
      format: {
        type: "json_schema",
        schema: {
          type: "object",
          properties: {
            right: { type: "string" },
            articles: { type: "array", items: { type: "string" } },
            plain: { type: "string" },
            steps: { type: "array", items: { type: "string" } },
            where_to_go: { type: "array", items: { type: "string" } },
            disclaimer: { type: "string" },
          },
          required: ["right", "articles", "plain", "steps", "where_to_go", "disclaimer"],
          additionalProperties: false,
        },
      },
    },
    messages: [{ role: "user", content: question }],
  });

  const text = response.content.find((b) => b.type === "text")?.text ?? "{}";
  return Response.json(JSON.parse(text));
}
```

> Cursor поможет дописать фронтенд, который шлёт `fetch("/api/ask", { method: "POST", body })`
> и рисует карточку. **Важно: Ақнұр должна понимать каждую строку этого кода** (см. §6).

## 4. База знаний (`knowledge-base.json`) — контент = половина успеха

### 4.1 Схема одной записи

```json
{
  "id": "TK-36",
  "topic": "Испытательный срок",
  "article": "Статья 36 Трудового кодекса РК",
  "plain": "Испытательный срок устанавливается только в письменном договоре и не может быть больше [проверить срок]. Во время испытания у работника те же права, включая оплату труда.",
  "keywords": ["испытательный", "стажировка", "проверка", "испыталка"],
  "source_url": "https://adilet.zan.kz/rus/docs/K1500000414"
}
```

### 4.2 Темы для первой работы (девочки собирают, ~15–20 записей)

1. Трудовой договор обязателен в **письменной** форме — нельзя «на словах»
2. Испытательный срок — максимум и права во время него
3. Рабочее время и **оплата переработок** (сверхурочные)
4. Сроки выплаты зарплаты, что делать при **задержке**
5. Отпуск и выходные
6. **Увольнение** — основания, предупреждение, окончательный расчёт
7. Особенности для **несовершеннолетних** (14–18 лет)
8. Оформление через **Enbek.kz** (электронный трудовой договор)
9. **Куда жаловаться:** госинспекция труда, единый контакт-центр **1414**, прокуратура

> ⚠️ **Точность закона важна.** НЕ выдумывайте номера статей и сроки. Берите точный текст и
> номера статей с официального [adilet.zan.kz](https://adilet.zan.kz) (Трудовой кодекс РК) и
> с [Enbek.kz](https://www.enbek.kz). В поле `plain` — свой перевод «на человеческий»,
> в `article` + `source_url` — точная привязка к первоисточнику. Это работа для девочек
> параллельно с кодом Ақнұр.

Это ещё и **измеряемый социальный эффект** для TVEP: сколько реальных статей охвачено,
сколько типовых ситуаций закрывает продукт.

## 6. Cursor и AI-комплаенс — читать обязательно (это про баллы и дисквалификацию)

**Ответ Ақнұр про Cursor:** да, Cursor — нормальный, серьёзный редактор для кода (это
VS Code с ИИ). Регламент **прямо разрешает и даже приветствует** Cursor, Copilot, ChatGPT для
написания кода. Так что — вперёд, кодь через Cursor.

**НО — критично (из регламента, дословно):** команда обязана **полностью понимать структуру,
логику и архитектуру своего кода**, кто бы его ни написал — человек или ИИ. Если технический
член команды на Q&A/код-ревью **не может объяснить логику** своего или сгенерированного кода,
баллы за техническую защищённость (критерий финала №2) снижают **вплоть до 0**. Покупка
готового кода или плагиат — **немедленная дисквалификация**.

**Как кодить через Cursor правильно (практика):**
1. После генерации — **прочитай код** и попроси Cursor объяснить его («explain this»), пока не
   поймёшь каждую часть. Понимаешь → двигаешься дальше. Не понимаешь → не вставляй.
2. Делай **маленькими кусками** (компонент, один route, одна функция), а не «сгенерируй всё
   приложение». Маленькое проще понять и объяснить.
3. Веди код **сама руками через понимание**, а не копипастом гигантских блоков, которые не
   объяснишь на защите.
4. Умей объяснить своими словами: зачем нужен серверный route, почему ключ на сервере, что
   такое наша ИИ-цепочка (классификация → подбор статьи → ответ), почему мы ссылаемся на закон.

Если сможешь объяснить архитектуру из §3 своими словами на Q&A — критерий №2 у вас в кармане.

## 8. Соц-эффект и устойчивость (для TVEP — это соц-проект)

- **Измеряемый эффект:** N реальных статей ТК РК охвачено; M типовых ситуаций первой работы
  закрывается; на CustDev X из Y человек не знали своих прав до продукта. Метрика: «время от
  вопроса до понятного ответа со ссылкой на закон» ~30 секунд против «не знаю / гуглю часами /
  молчу».
- **Кому помогает:** молодёжь 16–23 без доступа к юристу, впервые выходящая на рынок труда.
- **План устойчивости:** контент (база знаний) расширяется на аренду/ИП/потреб-права той же
  архитектурой; партнёрство с вузами/центрами занятости/Enbek; открытый датасет «право простыми
  словами». Стоимость запросов к Claude минимальна (короткие ответы, дешёвый haiku на подбор).
